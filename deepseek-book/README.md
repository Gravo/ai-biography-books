## 第4章 技术架构：MoE与MLA创新

### 4.1 DeepSeekMoE架构

#### 与传统MoE对比

```
传统MoE (Mixtral等)          DeepSeekMoE
┌─────────────────┐          ┌─────────────────┐
│  N个专家均分    │          │  细粒度专家分割  │
│  Top-2路由      │          │  Top-K自适应路由 │
│  无共享专家     │          │  共享专家隔离   │
└─────────────────┘          └─────────────────┘
        │                            │
        ▼                            ▼
  问题：专家冗余             优势：专家差异化
  问题：负载不均             优势：负载自适应
  问题：扩展受限             优势：无限扩展
```

#### DeepSeekMoE核心创新

```python
# 传统MoE路由
expert_ids = topk(router(hidden), k=2)
output = sum(experts[i](hidden) for i in expert_ids)

# DeepSeekMoE路由（简化示意）
# 1. 细粒度分割：每个FFN分成m个细粒度专家
# 2. 共享专家：部分专家对所有token激活
# 3. 路由专家：剩余专家选择性激活

shared_expert_out = shared_expert(hidden)  # 始终激活
routed_expert_ids = topk(router(hidden), k=k_routed)
routed_expert_out = sum(routed_experts[i](hidden) for i in routed_expert_ids)
output = shared_expert_out + routed_expert_out
```

#### DeepSeekMoE优势

| 优势 | 说明 |
|------|------|
| **专家差异化** | 细粒度分割使专家更专业化 |
| **知识共享** | 共享专家避免重复学习 |
| **负载均衡** | 自适应路由避免专家过载 |
| **扩展性强** | 可无限扩展专家数量 |

### 4.2 Multi-Head Latent Attention (MLA)

#### 传统MHA的问题

```
传统Multi-Head Attention：
┌─────────────────────────────────────────────────────┐
│  每个Head独立存储K和V                                │
│  KV Cache = n_heads × d_head × seq_len × batch     │
│  例：128K上下文 × 32头 = 数GB显存                   │
└─────────────────────────────────────────────────────┘
```

#### MLA创新

```
Multi-Head Latent Attention：
┌─────────────────────────────────────────────────────┐
│  1. 将KV压缩到低维latent空间                         │
│  2. 推理时解压恢复                                   │
│  KV Cache = d_latent × seq_len × batch             │
│  压缩比：93%（从64维降到4维latent）                  │
└─────────────────────────────────────────────────────┘
```

#### MLA效果

| 指标 | 传统MHA | MLA |
|------|---------|-----|
| KV Cache大小 | 100% | **7%** |
| 推理速度 | 1x | **10x** |
| 支持上下文 | 32K | **128K** |
| 推理成本 | 高 | **低** |

### 4.3 FP8混合精度训练

DeepSeek-V3首次在超大规模模型上成功应用FP8训练：

```
精度对比：
┌──────────────┬────────────┬─────────────┐
│  精度        │  显存占用   │  稳定性     │
├──────────────┼────────────┼─────────────┤
│  FP32        │  100%      │  最高       │
│  FP16/BF16   │  50%       │  高         │
│  FP8         │  25%       │  中高       │ ← DeepSeek突破
│  INT8        │  25%       │  低         │
└──────────────┴────────────┴─────────────┘
```

#### FP8训练关键创新

1. **细粒度量化**：每512个token一组缩放因子
2. **混合精度策略**：关键层保持BF16
3. **动态缩放**：训练过程中自适应调整
