## 💡 三句总结（给读者的行动召唤）

### 第1句："质疑常识"

```
行动：
  → 列出你领域的**3个"常识"**
  → 问："这是**真的**吗？有证据吗？"
  → 如果"没有证据" → 你找到了"创新机会"

例子：
  → 常识1："AGI需要更大模型"（AI领域）
    - 质疑：人类大脑只有86B神经元，为什么需要1T参数的模型？
    - 创新机会："更高效架构"（Mamba、RWKV...）

  → 常识2："Scale is All You Need"（AI领域）
    - 质疑：如果"更大模型"真的能到AGI，为什么GPT-4不能"理解物理"？
    - 创新机会："因果推理架构"（Neuro-Symbolic AI）

  → 常识3："Transformer是终极架构"（AI领域）
    - 质疑：Transformer的O(n²)复杂度真的不可接受吗？
    - 创新机会："线性复杂度架构"（State Space Models）
```

### 第2句："从第一性原理出发"

```
行动：
  → 遇到问题时，问："**本质**是什么？"
  → 不要从"别人的方案"出发（类比思维）
  → 要从"基础真理"出发（第一性原理思维）

例子：
  → 问题："如何提高机器翻译质量？"
    - 类比思维："改进LSTM"（别人怎么做，我也怎么做）
    - 第一性原理思维：
      1. 本质："建模词与词的关系"
      2. 检验现有方案："LSTM能建模关系吗？能，但慢"
      3. 寻找替代："Attention能建模关系吗？能，且快"
      4. 结果：Transformer

  → 问题："如何构建AGI？"
    - 类比思维："堆更多数据、更大模型"（GPT路线）
    - 第一性原理思维：
      1. 本质："理解世界 + 行动"
      2. 检验现有方案："GPT能'行动'吗？不能（只能生成文本）"
      3. 寻找替代："需要'多模态 + 机器人控制'"
      4. 结果：Adept AI的ACTION模型（？）
```

### 第3句："做减法，不是加法"

```
行动：
  → 遇到复杂方案时，问："能**去掉**这个组件吗？"
  → 如果去掉后性能只降1-2% → **去掉它**！
  → 简单架构 > 复杂架构（易并行、易调试、易泛化）

例子：
  → Transformer：**去掉**RNN → 更简单、更快
  → Mamba：**去掉**Attention → 更简单、更快
  → KAN：**去掉**MLP → 更简单、更易解释

反例：
  → GPT-4：在GPT-3上**加**更多层、更多数据 → 更复杂、更慢、更贵
  → 结果：只有**大公司**（OpenAI、Google、Meta）能玩得起
```
